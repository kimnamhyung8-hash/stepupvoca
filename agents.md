# 에이전트 행동 지침

## 언어 설정
- 모든 대화와 설명은 **한국어**로 진행한다.
- 코드 내 주석은 프로젝트 표준에 맞추되, 설명이 필요한 경우 한국어로 병기한다.

## 코드 수정 규칙 (중요!)
- 코드를 대량으로 삭제하거나 수정하기 전에 반드시 변경 사항의 이유를 한국어로 먼저 설명하라.
- 사용자의 승인 없이 파일 전체를 다시 쓰지 마라.
- 'Shadow Refactoring' 기능을 실행할 때 기존 로직이 유실되지 않도록 극도로 주의하라.
- 복잡한 로직은 수정 전 백업 주석을 남기거나 단계별로 진행하라.
## UI 및 메뉴 수정 제한 (절대 준수!)
- 사용자가 명시적으로 수정을 지시하지 않은 메뉴, UI 컴포넌트, 디자인 요소(뱃지, 애니메이션, 색상 등)는 절대 임의로 건드리지 마라.
- 기능 구현 시 해당 기능과 직접적으로 관련 없는 주변 UI에 영향을 주지 않도록 극도로 주의하라.

## 작업 시작 전 필수 체크
- 모든 작업 시작 전 반드시 이 `agents.md` 파일을 읽고 지침을 리마인드한 뒤 개발에 착수하라.

## ⚡ 하이브리드 (Cross-Platform) 코드 안전 원칙 (가장 먼저 고려할 것)
- `src/` 폴더 내의 공용 프론트엔드/로직 코드를 수정할 때는 특정 플랫폼(iOS/Android)만의 버그를 잡기 위해 타 플랫폼 시스템을 붕괴시키는 일이 없도록 극도로 주의하라.
- 플랫폼 종속적인 수리가 필요할 때는 반드시 **기기 방어벽 (If문 분기 처리를 통한 플랫폼 확인)**을 설계한 뒤 수리하라.
- (예시) "어? 만약 지금 돌아가는 기계가 아이폰(`Capacitor.getPlatform() === 'ios'`)이면 이 수술 코드를 실행하고, 안드로이드나 웹이면 원래 흐름대로 냅둬!" 라는 명확한 의도를 코드 구조에 반영할 것.

## 🔄 멀티 에이전트 동기화 (Sync) 워크플로우
- **작업 시작 전:** Windows와 Mac의 양방향 개발 체계를 유지하기 위해, 에이전트는 사용자와의 작업 세션(혹은 대화)을 시작하기 전에 가장 먼저 터미널을 통해 깃허브에서 최신 코드를 다운로드(`git pull`)하여 현재 로컬 저장소 상태를 완벽하게 최신화하라. 이를 통해 변경점을 파악하고 안전하게 작업을 설계할 수 있다.
- **작업 완료 후:** 코딩 및 주요 테스트가 종료되면, 즉시 사용자의 승인을 받거나 함께 변경 사항이 무엇인지 명확하게 체크할 수 있도록 커밋 메시지를 상세히 작성하여 깃허브에 업로드(`git push`)하라. 상대방(Mac/Windows)의 에이전트가 바로 이어서 일할 수 있도록 빈틈없이 연결하라.
